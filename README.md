🚀 Exploring the Power of AWS Lambda!

I recently built a serverless project using AWS Lambda that fetches metadata from any file URL — including content type and size — without ever provisioning a server. This project highlights how serverless computing allows developers to focus on code, not infrastructure, and instantly scale with demand.

🔹 Technologies used: AWS Lambda, API Gateway, Python, Docker, Amazon ECR
🔹 Key learnings: Handling HTTP requests in Lambda, working with Dockerized serverless applications, parsing JSON payloads, and integrating Lambda with API Gateway

Here’s what I accomplished:

Docker: Wrote a Dockerfile, built the image, and ran it locally.
Amazon ECR: Created a private registry and pushed the Docker image.
AWS Lambda (container image): Deployed the function directly from a Docker image instead of a zip file.
API Gateway: Exposed the function as a real public HTTP endpoint.
AWS CLI: Authenticated, managed resources, and tested everything from the terminal.
Serverless architectures are changing the way we build applications — this project was an exciting dive into that future!
